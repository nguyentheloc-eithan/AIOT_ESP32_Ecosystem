## 1. Project Overview

The Smart Garden system is an IoT-based automated irrigation solution designed to monitor and control garden watering remotely. The system combines a web-based interface, a cloud backend, and embedded hardware to give users full visibility and control over their garden's health — from anywhere, at any time.

**Core Goals:**

- Eliminate manual watering through scheduled automation
- Enable real-time remote control via a mobile-friendly web app
- Proactively alert the owner when soil conditions require attention

---

Stage 1 focuses on three core capabilities:

| Capability              | Description                                                                                                                |
| ----------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| **Schedule Management** | Create and manage fixed watering schedules with configurable times and durations                                           |
| **Manual Override**     | Instantly turn the water pump on or off from the web app, with real-time sync to the physical device                       |
| **Smart Notifications** | Push alerts to the owner's phone when soil moisture drops below a critical threshold outside of scheduled watering windows |

---

## 3. Features

### 3.1 Watering Schedules

Users can configure recurring watering times (e.g. 6:00 AM and 6:00 PM) with a set duration per session (e.g. 5 minutes). All schedule data is stored centrally in the cloud to ensure it persists through power outages or device restarts.

### 3.2 Manual Control

A prominent toggle or button in the web interface allows the user to activate or stop the pump immediately. Any action taken on the web app is reflected in real time on the physical device's display, and vice versa — the system stays in sync across all surfaces.

### 3.3 Smart Notifications

The system continuously monitors soil moisture levels. If moisture falls below 20% (dry threshold) and no scheduled watering session is currently active, the system automatically sends a push notification to the owner's phone.

**Example notification:**

> _"Your garden is very dry (Moisture: 15%). Would you like to water now?"_

## 4. System Architecture

The system is composed of three layers that communicate in a defined flow:

```
Hardware (M5Stack)
    │  Reads sensors → Publishes data via MQTT
    ▼
Backend (Golang)
    │  Subscribes to MQTT → Updates database
    │  Runs scheduled jobs → Triggers pump commands
    │  Monitors moisture → Sends push notifications (FCM)
    ▼
Frontend (React PWA)
       Displays live data → Sends control commands
       Receives push notifications on mobile
```

**Key Integrations:**

| Service                            | Role                                             |
| ---------------------------------- | ------------------------------------------------ |
| **MQTT Broker**                    | Real-time messaging between hardware and backend |
| **Supabase / PostgreSQL**          | Persistent storage for sensor logs and schedules |
| **Firebase Cloud Messaging (FCM)** | Push notifications to the owner's mobile device  |

## 6. Physical Enclosure

All components are housed in a single waterproof enclosure for a plug-and-play installation:

- **Lid:** The M5Stack Core2 is mounted flush with the lid so the display is visible without opening the box.
- **Inside:** The 12V power supply and a step-down converter (12V → 5V) are mounted internally to power the M5Stack.
- **External connectors:**
  - 1× AC 220V power inlet
  - 1× DC output to the solenoid valve
  - 1× Grove port extension for the external moisture sensors

---

## 7. Data Model

The system uses two primary data tables:

**Sensor Logs** — records timestamped moisture readings from the sensors.

**Watering Schedules** — stores user-defined watering sessions, each with a start time, duration, and active/inactive status.
